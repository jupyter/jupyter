# Contributing to Jupyter Inquiry Labors

Thank you for considering contributing to this template repository!

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker
- Check if the issue already exists
- Provide clear description and steps to reproduce
- Include environment details if relevant

### Suggesting Enhancements

- Open an issue with the "enhancement" label
- Clearly describe the enhancement and its benefits
- Provide examples if possible

### Adding New Templates

To add a new notebook template:

1. **Create the notebook** in `notebooks/templates/`
2. **Follow naming convention**: `##_descriptive_name_template.ipynb`
3. **Include all essential elements**:
   - Header with title, author, date
   - Section headers with markdown
   - Code cells with comments
   - Example usage patterns
   - Summary/conclusion section

4. **Document the template** in `notebooks/README.md`:
   - Add description
   - List key features
   - Explain use cases

5. **Test the template**:
   - Ensure all imports work
   - Verify code cells run without errors
   - Check markdown renders correctly

### Improving Documentation

- Fix typos and grammatical errors
- Add examples and clarifications
- Update outdated information
- Improve organization

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test your changes**
5. **Commit with clear messages**: `git commit -m "Add feature: description"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Open a Pull Request**

## Style Guidelines

### Notebook Style

- Use clear, descriptive titles
- Add markdown documentation between code sections
- Keep code cells focused and concise
- Include comments for complex logic
- Use consistent formatting

### Python Code Style

- Follow PEP 8
- Use meaningful variable names
- Add docstrings to functions
- Include type hints where appropriate
- Keep functions focused and modular

### Documentation Style

- Use clear, concise language
- Include examples
- Break complex topics into sections
- Use proper markdown formatting
- Keep tables and lists organized

## Pull Request Process

1. **Update documentation** if needed
2. **Add tests** for new functionality
3. **Ensure all tests pass**
4. **Update the README.md** with details of changes if applicable
5. **Request review** from maintainers

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or insulting comments
- Publishing private information
- Other unprofessional conduct

## Questions?

Feel free to open an issue with your question or reach out to the maintainers.
